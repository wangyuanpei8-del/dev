# 输出（本项目路径 `test/`，勿用 D:\AItest\case\）

**同时生成三个文件**（内容一致）：

| 文件 | 路径 | 工具 |
|------|------|------|
| Markdown | `test/寮ProjectWyp_{随机5位}.md` | Write 或 `scripts/generate-dom-test-cases.mjs` |
| JSON | `test/寮ProjectWyp_{随机5位}.json` | Write |
| Excel | `test/寮ProjectWyp_{随机5位}.xlsx` | Shell: `node scripts/test-cases-to-xlsx.mjs test/寮ProjectWyp_{随机5位}.json` |

随机数：5 位数字或 `yyyyMMdd-HHmm`。**三个文件必须使用同一后缀。**

**不要** 调用 MCP `excel_create_table`（本项目未配置）；**必须** 用仓库内 `scripts/test-cases-to-xlsx.mjs` 生成 Excel。

# Excel / JSON 格式规范（8 列固定）

| 列 | JSON 字段 | 说明 |
|----|-----------|------|
| A | `module` | 所属模块（如「入退寮管理」） |
| B | `requirement` | PRD 章节或 F-xxx（如「F-031 入寮登録」） |
| C | `name` | `[条件]_动宾结构` |
| D | `preconditions` | 数字编号，`\n` 换行 |
| E | `type` | 功能测试/接口测试/鲁棒性测试/安全测试/兼容性测试 |
| F | `steps` | **见下方操作类型规范** |
| G | `expected` | 与步骤一一对应，数字编号 |
| H | `priority` | P1–P4 |

2. **用例汇总表**（模块 × P1–P4 计数）

4. 步骤列保留 `[操作类型]` 标签

# 工作流程

## 第一步：需求分析

1. 读取 `dom-dev/doc/详细设计`、`func_front.md` 相关章节
2. 列出模块清单（路由 + DB 表 + 已有 E2E 见 `docs/stage-8-test-design.md`）
3. 标注隐含需求：乐观锁 version、RBAC、性别/重叠业务规则、软删除

## 第二步：按模块设计用例

**每个模块至少 5 条**，覆盖：

| 类别 | 最少 | 说明 |
|------|------|------|
| 正常路径 | 1 | 页面加载、主流程 |
| 新增/创建 | 1 | 向导或 Dialog |
| 异常路径 | 1 | 错误凭证、无权限 403 |
| 边界值 | 1 | 空搜索、必填校验 |
| DB 验证 | 1 | 对应 PostgreSQL 表 |

## 第三步：写入交付物

1. 生成完整 `cases` 数组 → Write JSON
2. 根据 JSON 生成 Markdown（表格与 JSON 一致）
3. Shell 执行：`node scripts/test-cases-to-xlsx.mjs test/寮ProjectWyp_xxxxx.json`
4. 确认三个文件存在且条数一致

## 第四步：交付总结

```
本次已针对 寮ProjectWyp（社員寮管理），
完成对 {N} 个模块的测试用例设计，共 {M} 条，
已保存至 test/：
  - 寮ProjectWyp_{id}.md
  - 寮ProjectWyp_{id}.json
  - 寮ProjectWyp_{id}.xlsx
可追溯：dom-dev/doc/详细设计；前端索引：func_front.md
E2E 冒烟：npm run test:web
```
