## 3.3 功能详述（节选）

### F-040 入居登记

**为什么做**：核心业务，替代 Excel 入居行。

**用户价值**：自动防止同室双人、性别错误。

**边界条件**：

- 入寮日 ≤ 退寮日（若已填）
- 同一 `room_id` 下区间 `(start, end)` 与既有履歴不重叠（**闭区间**：退寮日当天算在室；退寮日为空视为无限期）
- 社員 `gender` 与 `dorm.gender_type` 一致
- 1 户最多 3 人：同一 `dorm_id` 在同一日期的在室人数 ≤ 3

**错误场景**：

| 场景 | 错误码 | 消息 |
|------|--------|------|
| 期间重叠 | `OCCUPANCY_OVERLAP` | 该部屋在该期间已有入居者 |
| 性别不符 | `GENDER_MISMATCH` | 社員性别与寮种别不一致 |
| 超过定员 | `DORM_CAPACITY_EXCEEDED` | 该户在室人数已达上限 |

### F-042 初回入寮日

**规则**：

- 仅 `employee_type = JAPAN` 时，在**首次**创建入居履歴且 `first_dorm_use_date` 为空时，设为本次 `move_in_date`
- 部屋移动、退寮后再入居：**不更新** `first_dorm_use_date`
- `employee_type = CHINA_ASSIGNMENT` 字段保持 `NULL`

### F-050 寮費月次算定

**输入**：`employee_id`, `year_month`（如 2026-04）

**计算**：

```text
amount = round( room_area_m2 * daily_rate(room_type) * occupied_days / days_in_month , 0 )
```

- `occupied_days`：该月在室日历日数（入退寮按日割り）
- 保存 `calculation_basis` JSON：`{ area, room_type, daily_rate, occupied_days, days_in_month }`

**边界**：未入居月份不可算定；已「确定」月份不可覆盖（Admin 除外）。

### F-060 空き室判定

部屋 `R` 在查询日 `D` 为空：

1. 不存在 `move_in_date <= D` 且 (`move_out_date` IS NULL OR `move_out_date >= D`) 的履歴；且  
2. 不存在 `move_in_date > D` 的将来予定入居。

> **在室判定（重要统一口径）**：在查询日 `D`，满足 `move_in_date <= D` 且 (`move_out_date` IS NULL OR `move_out_date >= D`) 视为在室。  
> 空室判定与在室判定必须使用同一口径（与最新要件定義書一致）。

## 3.4 非功能需求

| 类别 | 要求 |
|------|------|
| 性能 | 一覧 API P95 < 500ms（20 并发、约 15 户规模） |
| 并发 | 约 20 用户同时操作 |
| 保留 | 业务数据 5 年以上；逻辑删除 |
| 安全 | HTTPS、bcrypt、JWT 短期、CORS 限制、审计日志 |
| 可用 | MVP 单实例 + 每日 DB 备份 |
| 语言 | UI 默认日语；错误消息日语 |
