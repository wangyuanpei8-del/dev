# Stage 1：项目立项说明书

## 1.1 项目背景

公司约 15 户员工寮（以 3DK 为主）目前依赖 Excel 管理入居者、部屋、寮費及长期利用信息。混在日本社員与中国出張社員、男女别寮、入退寮历史与「初めて寮を使用した入寮日」等规则下，手工表格易出错、难以审计，且无法支撑长期利用警告与値上げ决策。

本项目在**生成 AI 开发评估**前提下，建设可运行的 Web 管理系统，并为将来本番化预留扩展空间。

## 1.2 项目目标

| 目标 | 说明 |
|------|------|
| G1 一元管理 | 寮・部屋・入退寮・寮費・空き室在同一系统管理 |
| G2 规则内置 | 男女混住禁止、入居重叠禁止、性别匹配等由系统强制 |
| G3 长期利用可视 | 基于初回入寮日计算通算期间，支持 3 年超警告与値上げ参考 |
| G4 Excel 脱却 | 支持现有 Excel 初始导入，日常运营不再依赖表格 |
| G5 可追溯 | 操作ログ、逻辑删除、5 年以上履历保留 |

## 1.3 KPI（MVP 验收后 3 个月内可衡量）

| KPI | 目标值 | 测量方式 |
|-----|--------|----------|
| Excel 依赖度 | 日常业务 0 Excel | 运营调查 |
| 入居冲突 | 0 件重叠/性别违规数据 | DB 校验脚本 + 审计 |
| 数据迁移 | ≥95% 历史行成功导入 | 导入结果报表 |
| 画面操作 | 主要流程 ≤5 次点击完成入居登记 | 可用性走查 |
| 可用性 | 工作时间可用性 ≥99%（单实例 MVP） | 监控 |

## 1.4 用户价值

| 用户 | 价值 |
|------|------|
| 総務・寮管理担当 | 减少手工、降低误操作、快速查空室与长期利用者 |
| 管理层 | 长期利用与寮費数据可导出，支撑値上げ判断 |
| IT / 开发 | 结构化数据与 API，便于与 HR 等系统将来集成 |

## 1.5 商业价值

- 降低合规与性别混住风险
- 为寮費规则变更（面积・日割り・社员区分）提供可配置基础
- 评估生成 AI 在全流程 SDLC 中的可交付性

## 1.6 范围（In Scope — MVP）

1. 认证与 RBAC（本地账号）
2. 社員マスタ（区分：日本 / 中国出張）
3. 寮・部屋マスタ（男性寮 / 女性寮）
4. 入退寮履歴（含退寮理由、业务约束校验）
5. 初めて寮を使用した入寮日（日本社員のみ）
6. 寮費算定与月次记录（面积・期间日割り）
7. 空き室一覧（寮单位 / 部屋单位、性别过滤）
8. Excel 导入（寮・部屋・入居履歴・初回日・寮費参考值）
9. 操作ログ
10. 备品マスタ、保管场所（基础 CRUD）

## 1.7 非范围（Out of Scope — MVP）

| 项目 | 说明 |
|------|------|
| 入居时备品准备 | 要件 7.2 明确本期不做 |
| 退去备品废棄/保管流程 | Phase 2 |
| HR 系统实时同步 | Phase 2 |
| 邮件/Slack 自动通知 | Phase 2 |
| 移动 App | 不需要 |
| 支付/请求书发行 | 不需要 |
| 多租户 / 多公司 | 不需要 |

## 1.8 里程碑（建议）

| 阶段 | 周期 | 交付 |
|------|------|------|
| M1 基础 | Week 1-2 | Monorepo、DB、认证、寮/部屋/社員 CRUD |
| M2 核心 | Week 3-4 | 入退寮、约束、初回日、空き室 |
| M3 费用 | Week 5 | 寮費算定、月次一覧 |
| M4 导入 | Week 6 | Excel 导入 + 迁移验证 |
| M5 上线 | Week 7 | 测试、Docker、部署、培训 |

## 1.9 成功标准

- 要件定义书中 MVP 范围内功能均可演示
- 从现有 Excel 完成一次完整导入且可继续日常录入
- 入居约束违规在 API 层被拒绝并返回明确错误码
- 文档与代码目录与 `stage-0` ~ `stage-9` 一致，可供 Cursor 继续生成 Stage 7 代码
