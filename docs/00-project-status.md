# 当前项目状态（Project Status）

> 每次进入新开发阶段前请先更新本文件，禁止与已发布文档冲突。

## 已确认

| 项目 | 内容 |
|------|------|
| 系统名称 | 寮管理システム（社員寮管理） |
| 对象规模 | 约 15 户；多为 3DK（3 个单间），1 户最多 3 人 |
| 所在地 | `TOKYO / OSAKA / NAGOYA / OTHER`（枚举） |
| 中心画面 | **寮割カレンダー**（`/allocation-calendar`） |
| 寮种别 | 男性寮 / 女性寮；同一户禁止男女混住 |
| 入居者区分 | 日本社員（正社員・契約社員）、中国出張社員 |
| 核心业务 | 寮割・入退寮・寮費・空き室・責任者★・所属マスタ・CSV/印刷・操作ログ |
| 在室口径 | **闭区间**：退寮日当天算在室；NULL 退寮=无期限 |
| 入居约束 | 同室**日粒度**禁止多人；期间重叠禁止；性别与寮种别必须一致 |
| 初回入寮日 | 仅日本社員；中国出張社員除外；用于 3 年长期利用警告与値上げ判断 |
| 非功能 | 并发约 20 人；履历保留 5 年以上；逻辑删除 |
| MVP 除外 | **入居时备品准备（要件 7.2）本期不开发** |
| 技术栈 | Vue3 + Vite + Element Plus + NestJS + Prisma + PostgreSQL + Monorepo（见 Stage 0） |

## 技术决策

| 决策 | 选型 | 理由 |
|------|------|------|
| 架构 | Monorepo `apps/web` + `apps/api` | 与 SDLC 默认栈一致，便于全栈交付 |
| API | REST + JSON 统一包装 | NestJS 生态成熟，Cursor 易生成 |
| DB | PostgreSQL + UUID + snake_case + soft delete | 符合总提示词规范 |
| 业务约束 | Service 层 + DB 部分唯一约束 | 入居重叠、性别校验必须在服务端强制执行 |
| 前端 | Vue3 + Vite + Element Plus + Axios + Echarts（PC 后台，见 01-前端开发工程师） |
| Excel | 服务端 `xlsx` 解析 + 预览 + 事务导入 | 满足要件 9 |

## 风险

| 风险 | 影响 | 缓解 |
|------|------|------|
| Excel 列名与现物不一致 | 导入失败率高 | 提供列映射 UI + 模板下载 |
| 入居期间重叠逻辑复杂 | 数据不一致 | 单元测试覆盖边界日期；DB exclusion 或应用层锁 |
| 日割り寮費与历史费率变更 | 金额争议 | 寮費记录保存「算出根拠」JSON |
| 15 户规模虽小但规则多 | 低估工期 | 严格 MVP 范围，备品准备延后 |
